Ingredients:

Canvas Button Classes with Data Attributes (Recommended)
  • Element: <a> tag with Canvas button classes
  • Classes: "btn-large btn" (Canvas built-in button styling)
  • Data attributes: data-course-type, data-published, data-api-endpoint, data-api-returntype
  • Custom colors: Override with inline styles for OSU branding
  • Arrow: → (right arrow character)
Alternative: Simple Styled Links
  • Element: <a> tag with inline styles only
  • Display: inline-block
  • Padding: 12px 24px (vertical, horizontal)
  • Border-radius: 6px
  • Font-size: 12pt
  • Font-weight: bold
  • Text-decoration: none (removes underline)

HTML/CSS Code:

Canvas Button Classes with Data Attributes (Recommended)
<!-- Assignment Button (Beaver Orange) -->
<p><a class="btn-large btn"
       style="background: #d73f09; color: white; padding: 12px 24px; border-radius: 6px; display: inline-block; font-size: 12pt;"
       href="https://canvas.oregonstate.edu/courses/YOUR_COURSE_ID/assignments/YOUR_ASSIGNMENT_ID"
       data-course-type="assignments"
       data-published="true"
       data-api-endpoint="https://canvas.oregonstate.edu/api/v1/courses/YOUR_COURSE_ID/assignments/YOUR_ASSIGNMENT_ID"
       data-api-returntype="Assignment"><strong>Submit Assignment →</strong></a></p>

<!-- Discussion Button (Secondary Style) -->

       data-course-type="discussions"
       data-published="true"

Suggested Uses:

  • Assignment submission links
  • Discussion board navigation
  • External resource access
  • Quiz and assessment entry points
  • File download prompts
  • Course navigation elements

Design Guidelines:

  • Use Canvas Button classes for interactive elements (show/hide content, expand sections, etc.)
  • Use styled links for navigation to Canvas pages, assignments, discussions, and external resources
  • Important: Canvas Button classes create buttons, not links - they won't navigate to other pages. You can assign the button classes to an anchor tag: a class="btn-large btn"
  • Use primary (orange) buttons for main actions
  • Use secondary (outlined) buttons for alternative actions
  • Include directional arrow (→) for navigation clarity
  • Center buttons in containers for emphasis
  • Maintain consistent padding and sizing
  • Use descriptive, action-oriented text

Accessibility Features:

  • High contrast color combinations (AA 2.1 compliant)
  • Large click target (minimum 44px height)
  • Clear, descriptive link text
  • Keyboard accessible with proper focus states
  • Works with screen readers

Canvas Integration:

  • data-course-type: "assignments", "discussions", "files", "quizzes", "modules", etc.
  • data-published: "true" for published content, "false" for unpublished
  • data-api-endpoint: Full Canvas API URL for the resource
  • data-api-returntype: "Assignment", "Discussion", "Folder", "Quiz", etc.
  • href: Direct Canvas URL to the resource (what users will navigate to)
  • Canvas URL patterns: /courses/ID/assignments/ID, /courses/ID/discussion_topics/ID, /courses/ID/files

Implementation Steps:

  1. In Canvas Rich Content Editor, click the "HTML Editor" link (usually in the bottom right)
  2. Copy the HTML code from the examples above
  3. Paste the code into the HTML editor
  4. Replace placeholder URLs and IDs with your actual Canvas resource URLs
  5. Switch back to the visual editor and save to see the styled buttons
Canvas Compatibility: Requires HTML editor access in Canvas Rich Content Editor
Dark Mode: High contrast maintained in Canvas dark mode
