Standard Page Structure

1. Page Header

Every page begins with a standardized header section that includes the component category title and brief description.

Required Elements:
  • H2 heading (16pt, #333 color) - Main component category
  • Subtitle paragraph (#333 color for accessibility) - Brief description
  • Border styling (3px solid #dc4405) - Consistent visual separation

2. Component Sections

Each individual component follows a consistent structure with header and body sections.

Component Structure:
  • Component Header with name and description
  • Preview Section with live demonstration
  • Ingredients Section listing CSS properties
  • HTML/CSS Code Section with copy-paste ready code
  • Suggested Uses Section with practical applications

Accessibility Requirements

Critical: All components must meet AA 2.1 accessibility standards
  • Text contrast: Minimum 4.5:1 ratio for text under 18pt (or bold 14pt)
  • Proper heading hierarchy: No skipping levels (h2→h3→h4→h5)
  • Semantic HTML: Proper heading structure, list markup, blockquote elements
  • Subtitle text: #333 minimum for sufficient contrast
  • Body text: #333 or darker for sufficient contrast

Component Evaluation Framework

Design Principles

  • Contrast: Clear visual hierarchy
  • Repetition: Fits existing design system
  • Alignment: Clean organization
  • Proximity: Logical grouping

Pedagogical Value

  • Enhances learning objectives
  • Appropriate for Canvas environment
  • Fills gap in existing library
  • Simple for instructors to use

Pro Tip: Test all components in Canvas RCE before publishing. Some formatting may change when edited in Canvas, so always verify the final appearance.
