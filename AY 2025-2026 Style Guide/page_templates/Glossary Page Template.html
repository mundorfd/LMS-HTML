Placeholder Guide:

  • [DECORATIVE-ICON-URL]: URL to a relevant decorative icon (75x74px recommended)
  • [ICON-API-ENDPOINT]: Canvas API endpoint for the icon file
  • [GLOSSARY TITLE]: Main title for the glossary (e.g., "Glossary of Cryptographic Abbreviations")
  • [MODULE/SECTION NAME]: Context identifier (e.g., "Module 8", "Chapter 3")
  • [TABLE CAPTION FOR SCREEN READERS]: Descriptive caption for accessibility
  • [COLUMN 1 HEADER]: First column header (e.g., "Abbreviation", "Term")
  • [COLUMN 2 HEADER]: Second column header (e.g., "Full Term", "Category")
  • [COLUMN 3 HEADER]: Third column header (e.g., "Explanation", "Definition")
  • [TERM 1-5]: Replace with actual terms, abbreviations, or concepts
  • [FULL NAME/EXPANSION 1-5]: Complete names or expansions of terms
  • [DEFINITION/EXPLANATION 1-5]: Clear, concise definitions or explanations
  • [SUMMARY NOTE]: Brief note about the glossary's purpose and scope

Content Organization Guidelines:

  • Alphabetical Order: Arrange terms alphabetically for easy reference
  • Consistent Formatting: Use bold for terms, italics for expansions
  • Clear Definitions: Write concise, jargon-free explanations
  • Appropriate Length: Keep definitions comprehensive but scannable
  • Cross-References: Link related terms when helpful
  • Context Relevance: Focus on terms specific to the module/course

Table Structure Options:

  • Three-Column Layout: Term | Full Name | Definition (recommended for abbreviations)
  • Two-Column Layout: Term | Definition (for simple glossaries)
  • Four-Column Layout: Term | Category | Definition | Example (for complex terminology)
  • Column Width Adjustments: Modify percentages based on content needs

Suggested Uses:

  • Course-specific terminology and abbreviation lists
  • Technical vocabulary for specialized subjects
  • Key concept definitions for complex modules
  • Reference pages for acronym-heavy content
  • Study aids and quick reference materials
  • Standardized terminology across course sections

Accessibility Features:

  • Semantic table structure with proper headers and scope attributes
  • Hidden caption for screen readers describing table content
  • Canvas ic-Table classes for consistent styling and behavior
  • High contrast colors meeting AA 2.1 standards
  • Decorative image properly marked with data-decorative attribute
  • Logical reading order and clear content hierarchy

Canvas Integration Notes:

  • Responsive Design: Table adapts to different screen sizes
  • API Integration: Includes proper data attributes for file references
  • Hover Effects: Built-in row highlighting for better usability
  • Print Friendly: Clean layout that works well in printed materials
