Enhanced Definition List

Styled definition list with colored terms and subtle visual hierarchy

Preview:

0 = No Problem
I don't agree that this is a usability problem at all.
1 = Cosmetic Problem Only
This does not need to be fixed unless extra time is available on the project.
2 = Minor Usability Problem
Fixing this issue should be given low priority.
3 = Major Usability Problem
It is important to fix this issue, so it should be given high priority.

Ingredients:

  • semantic HTML: Uses proper <dl>, <dt>, <dd> structure
  • term color: #d73f09 (Beaver Orange)
  • term font-weight: bold
  • term font-size: 14pt
  • term border-bottom: 1px solid #e6e7e8
  • term padding-bottom: 0.25rem
  • term margin: 1.5rem 0 0.5rem 0
  • definition margin: 0 0 1rem 1.5rem (indented)
  • definition font-size: 12pt, line-height 1.6

HTML/CSS Code:

<dl style="margin: 2rem 0;">
    <dt style="color: #d73f09; font-weight: bold; font-size: 14pt; margin: 1.5rem 0 0.5rem 0; border-bottom: 1px solid #e6e7e8; padding-bottom: 0.25rem;">Your Term</dt>
    <dd style="margin: 0 0 1rem 1.5rem; font-size: 12pt; line-height: 1.6;">Your definition or explanation goes here.</dd>
    
    <dt style="color: #d73f09; font-weight: bold; font-size: 14pt; margin: 1.5rem 0 0.5rem 0; border-bottom: 1px solid #e6e7e8; padding-bottom: 0.25rem;">Second Term</dt>
    <dd style="margin: 0 0 1rem 1.5rem; font-size: 12pt; line-height: 1.6;">Second definition or explanation.</dd>
</dl>

Suggested Uses:

  • Course glossaries and vocabulary definitions
  • Technical specifications and requirements
  • Step-by-step process explanations
  • Assessment criteria and rubric definitions
  • FAQ sections with structured questions and answers
Accessibility Note: Definition lists provide excellent semantic structure for screen readers and assistive technologies

Usage Guidelines

Best practices for implementing definition lists effectively

When to Use Definition Lists:

  • Term/Definition Pairs: Any content with clear term-explanation relationships
  • Instead of Bold-in-Bullets: More semantic than using bold text in regular lists
  • Glossaries: Course vocabulary and technical terminology
  • Specifications: Technical requirements, criteria, or parameters
  • Q&A Sections: Structured question and answer content

Accessibility Benefits:

  • Screen Reader Support: Assistive technologies understand the term/definition relationship
  • Semantic Structure: Proper HTML markup improves navigation
  • Keyboard Navigation: Users can jump between terms efficiently
  • Clear Hierarchy: Visual and structural organization aids comprehension

Maintenance Tips:

  • Copy Full Blocks: Always copy the complete <dl> structure when creating new lists
  • Consistent Styling: Use the same color theme throughout a section or document
  • HTML Editor Required: Definition lists must be created in Canvas HTML editor
  • Test Responsiveness: Preview on different screen sizes, especially horizontal lists
Usage Tip: Choose enhanced vertical lists for detailed explanations and horizontal lists for quick reference materials. Use color themes to coordinate with other course components.
